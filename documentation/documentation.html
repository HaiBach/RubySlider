QUICK START

Quick start and options will give the basics to run slider. The other sections will provide more detailed options.

1. Upload code folder to your host, this folder included core script, style, and image of codeslider to work

  • code

Other folder for reference

  • documentation | this user manual
  • example | basic example for codeslider
  • template | use codeslider in theme, more design and function
  • libs | js, css, image... → using in doc, example and template


2. Plugin requires jQuery 1.7+ :
If you do not have jQuery, you add it to your page

<!-- jQuery from Google hosted -->
<script src="//ajax.googleapis.com/ajax/libs/jquery/1.10.2/jquery.min.js"></script>


3. Add main js, css codeslider to your page:

<!-- Main style codeslider, included default skin -->
<link rel="stylesheet" href="code/codeslider.css">

<!-- Main js codeslider -->
<script src="code/codeslider.js"></script>


4. Add the slider HTML to container:

<div class="container">
    <!-- Codeslider HTML -->
    <div class="cs"> ... </div>
</div>
  • Width-slider always equal to width-container
  • Height-slider depend on each height-image background, more options → view height-type


5. Add slides to slider.

  • .cs-default → slider using default skin, build-in code/codeslider.css file.
  • Every child element will be treated as a slide, with any type tag
<div class="cs cs-default">
    <!-- Slide: simple image -->
    <img src="img.jpg" alt="[img]">

    <!-- Slide: image lazyload -->
    <a class="cs-img" href="img.jpg">[img lazyload]</a>
    
    <!-- Slide: image and layer -->
    <div>
        <a class="cs-img" href="img.jpg">[img background]</a>
        <h3 data-layer="...">This is layer</h3>
    </div>
</div>


6. Initialize the slider (usually at the end of the body section)

<script>
    jQuery(document).ready(function($) {
        $('.cs').codeslider();
    });
</script>
With few simple steps, you have created codeslider working!

Notes

  • Color of value type: number - string - boolean - mixed value

  • Options slider using both direct on slider via data-slider and javascript.
    Syntax in data-slider is string with formated [key]-[value], separated by space.
    Syntax in javascript is JSON formated. 

    <!-- Setup options direct on slider -->
<div class="cs" data-slider="speed-1000 isSlideshow-true"> ... </div>

<!-- Setup options in javascript -->
<script>
    jQuery(document).ready(function($) {

        $('.cs').codeslider({
            // options put here
            speed : 1000,
            isSlideshow : true
        });
    });
</script>
    View example

  • Options slide using direct on each slide via data-slide. slide-options will be priority than slider-options

    <div class="cs" data-slider="speed-400 delay-5000">
    
    <!-- slide custom setup -->
    <div data-slide="speed-600 delay-8000"> ... </div>
    ...
</div>
  • Options have is prefix → boolean value.
    true/on/1 same result for true boolean value in data-slider.
    false/off/0 same result for false boolean value in data-slider.

Image fit - fill

Each slide can be image background, and can be setting this fit or fill slide
Include 3 type:

  • imgWidth
  • imgHeight
  • img
<!-- Image width with autofit -->
<div class="cs" data-slider="imgWidth-autofit"> ... </div>

<!-- Image height with autofit -->
<div class="cs" data-slider="imgHeight-autofit"> ... </div>

<!-- Image width/height width autofill -->
<div class="cs" data-slider="img-autofill"> ... </div>
Image fit/fill and height-type have related intimately

imgWidth applied to all height-type, included settting:

  • smallfit : if width-img < width-slide → width-img = width-slide
  • largefit : if width-img > width-slide → width-img = width-slide
  • autofit : width-img = width-slide, with any width

imgHeight applied only to height-fixed, included settting:

  • smallfit : if height-img < height-slide → height-img = height-slide
  • largefit : if height-img > height-slide → height-img = height-slide
  • autofit : height-img = height-slide, with any height

img applied only to height-fixed, included settting:

  • smallfit | smallfill : if width/height-img < width/height-slide → image auto fit or in slide
  • largefit | largefill : if width/heght-img > width/height-slide → image auto fit or fill in slide
  • autofit | autofill : → image auto fit or fill in slide, width any width/height

Transition element

• All transition element in data-layer → simple, clean and flexible

• Element structure:
[key]-[value]

• Each transition include all elements:

Key Default value Description
x 0 Position horizontal
y 0 Position vertical
opacity 1 Layer opacity | value [0 - 1]
rotate 0 Transform rotate | unit 'deg'
scale 1-[1] Transform scale | include scaleX, scaleY
skew 0-[0] Transform skew | include skewX, skewY
start 400 Timer start begin transition | unit 'ms'
speed 400 Timer duration of transition | unit 'ms'
es ease Transition easing

Notes

  • Type color : number - string - mixed value
  • Value support:
    • Integer number → i.e. x-20
    • Float number → i.e. scale-2.5
    • Negative number → i.e. rotate--30
    • String → i.e. es-easeOutExpo
    • Two value → i.e. skew-10-30
  • scale: include scaleX, scaleY:
    [scale]-[xValue]-[yValue]
    i.e. scale-2-5 → scaleX = 2, scaleY = 5
    i.e. scale-1.5 → scaleX = 1.5, scaleY = 1.5
  • skew: include skewX, skewY → same scale
    i.e. skew-30 → scaleX = 30deg, scaleY = 0
View example

Multi transition

Initial position

Each layer has the initial position.

Layer with 1 transition :

  • → layer copy transform of transition → start transform with opacity-0 with start 0ms, speed 0ms
  • → new transform with start, speed in setup data-layer

i.e.

<!-- Layer 1: setup -->
<div data-layer="x-50 y-100 rotate-90 start-500 speed-1000" class="layer1">...</div>

<!-- Layer 1: transform chain -->
1. Transform: translate(50px, 100px) rotate(90deg), opacity(0), start 0ms, speed 0ms
2. Transform: translate(50px, 100px) rotate(90deg), opacity(1), start 500ms, speed 1000ms

Transition separation

Each layer supports multiply transition, each transition separated by  ; 

  • First transition value for initial position
  • No limit to number transition
<div
    data-layer="
        x-0 y-10 opacity-1 ;
        x-0 y-600 scale-2 start-500 speed-500 ;
        x-200 y-600 skew-30 start-500 ;
        x-200 y-300 rotate-90 start-500 ;
        ...
    "> ... </div>

Value element

Value element in this transition inherits properties other transition.

  • Begin, elements of transition, if not setup will get default value
  • Elements in next transition will inherit elements of last transition
  • In case initial position, it will inherit element next transition
  • Start value in next transition will counted when prev transition started
  • Opacity in intial position and last transition is 0
View example

Loop transition

Loop transition like css3 animation, support elements:

Key Default value Description
count 1 Number of times animation repeat.
Support integer number and 'infinite' value
direction normal whether or not the animation should play in reverse on alternate cycles.
Support 'normal' and 'alternate' value

Setup element in anywhere in data-layer :

<div
    data-layer="
        count-3 direction-alternate
        x-0 y-10 opacity-1 ;
        x-0 y-600 scale-2 start-500 speed-500 ;
        ...
    "> ... </div>
View example

Position other value

Position align

Codeslider support position align between layer and slide.
Each element x, y included 3 value:

  • x-[ left / center / right ]
  • y-[ top / center / bottom ]
View example

Position outside

Position outside slide are represented by element xy with 4 value :

  • xy-[ leftOut / rightOut / topOut / bottomOut ]

When setup position outside :

  • Layer auto calculation position x in case leftOut/rightOut, and position y will inherited by previous transition
  • Same for element y
  • In case initial position, x or y inherited next transition
  • Position ouside can't setup in continuous transition
View example

Random value

Each elements x | y | rotate | scale | skew → can be setup random value with value ra 

x-ra            // x random value between [0, width-slide]
y-ra            // y random value between [0, height-slide]
rotate-ra       // rotate random value between [-360deg, 360deg]
scale-ra        // scale random value between [0.2, 5]
scale-ra-ra     // scaleX scaleY random value
skew-ra         // skewX random value between [0deg, 90deg]
skew-ra-ra      // skewX, skewY random value
View example

RESPONSIVE

SLider will auto change width/height with ratio when width slider change.

Responsive setup

You only setting width value maximum to turn-on responsive.
Height auto get on image-background each slide.
responsive-[width] 

<!-- Example: maximum width is 1156px -->
<!-- Setting direct on slider -->
<div class="cs" data-slider="responsive-1156"> ... </div>
/* Or setting in script */
$slider.codeslider({
    responsive : 1156,
    ... 
});

If you want settup custom height fixed, smaller or larger than slide, add height value after width value
responsive-[width]-[height]

Add height value → option height-type auto convert to fixed 

<!-- Setting direct on slider -->
<div class="cs" data-slider="responsive-1156-600"> ... </div>
/* Or setting in script, value is string! */
$slider.codeslider({
    responsive : '1156-600',
    ... 
});
View example

Fullwidth

When setting option responsive , option fullwidth auto turn-on, unnecessary setting.
Make control slider in fullwidth easy, using options:

Canvas margin

Canvas is container all slide.
When responsive, width-slider smaller responsive-width, wanted margin canvas to slider, setting option:
mCanvas-[value] | unit px 

<!-- Setting direct on slider -->
<div class="cs" data-slider="responsive-1156 mCanvas-20"> ... </div>
/* Or setting in script */
$slider.codeslider({
    responsive : 1156,
    mCanvas    : 20,
    ... 
});

Media

In responsive, if you want setting width-slider from minimum to maximum value width of container or page, setting option:
media-[width]-[from]-[to]

  • If width into rank [form]-[to] on option media , margin canvas will auto convert value.
  • You can settup multi media
<!-- Setting direct on slider -->
<div class="cs" data-slider="
    responsive-940
    media-748-760-959   /* for tablet */
    media-300-0-479     /* for mobile */
    "> ... </div>
/* Or setting in script. If multi, value is array */
$slider.codeslider({
    responsive : 940,
    media      : ['748-760-959', '300-0-479'],
    ... 
});

  • In element, add class name to know type element.
